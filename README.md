# Retail Sales Analysis Using SQL

## Project Overview

This project analyzes retail sales transaction data using SQL to identify customer purchasing behavior, sales trends, and category performance. The analysis focuses on answering business questions related to revenue generation, customer activity, and sales patterns.

## Objectives

* Analyze overall sales performance.
* Identify top-performing customers.
* Evaluate product category performance.
* Understand customer demographics.
* Explore monthly and daily sales trends.
* Generate business insights using SQL.

## Tools Used

* SQL Server
* SQL Server Management Studio (SSMS)

## SQL Skills Demonstrated

* Data Cleaning
* Data Exploration
* Aggregate Functions (SUM, AVG, COUNT)
* GROUP BY and ORDER BY
* CASE Statements
* Common Table Expressions (CTEs)
* Window Functions (RANK)
* Date Functions
* Business-Oriented Data Analysis

## Analysis Performed

### Data Cleaning

* Checked for missing values
* Removed incomplete records
* Validated transaction data

### Sales Analysis

* Total sales transactions
* Sales by product category
* High-value transactions
* Monthly sales performance

### Customer Analysis

* Unique customer count
* Top customers by sales
* Average customer age by category
* Customer distribution across categories

### Operational Analysis

* Sales activity by shift
* Transaction patterns throughout the day
* Category-level customer engagement

## Key Insights

* Sales performance varies across product categories, with some categories generating significantly higher revenue than others.
* A small group of customers contributes a substantial portion of total sales, highlighting the importance of customer retention.
* Customer demographics differ across product categories, suggesting opportunities for targeted marketing.
* Monthly sales performance shows fluctuations throughout the year, indicating possible seasonal purchasing behavior.
* Sales activity is distributed differently across morning, afternoon, and evening shifts, providing insights for operational planning.
* Customer interest varies by category, with some categories attracting a larger number of unique customers.

## Conclusion

This project demonstrates how SQL can be used to clean, explore, and analyze retail transaction data to generate meaningful business insights. Through data exploration, aggregations, CTEs, and window functions, the analysis provides a better understanding of sales performance, customer behavior, and operational trends.
